    學員專題實戰手冊¶

    目標：從 0 到 1 建立「面試作品集」，並且升級成「業界最低標準等級的專案」

    學員專題大致可以分成三種樣貌：

    等級 樣貌 面試時的結果
    作業 跑得起 Jupyter Notebook、但沒架構圖、demo 只能手動執行 面試官比較難看到亮點
    作品 有敘述、能持續執行、技術棧豐富 「跟其他人差別不大」
    MVP 最小可行性產品 (demo) 有架構圖、有歷史紀錄，甚至能有監控和有下一步行動、能講為什麼選擇用 A 而不是選擇 B 想深入了解專案

    透過這裡幫你達到第三種等級

    先了解的 4 件事¶

    1. Data Pipeline 通用藍圖 — 所有 DE 專案的骨架是同一套，看完就再也不會「不知道從哪開始」
    2. 小組 10 階段流程 — 從第一次討論到 demo day 的完整 SOP
    3. 協作過程 — 敏捷開發、Git 版本控制、溝通工具、專案管理工具
    4. 8 情境完整 walkthrough — 8 種場景能照著走，並加以改造

    # 情境 對應產業 過往作品參考
    1 餐飲連鎖 POS 庫存優化 餐飲 / 零售 美味餐廳隨拾取、食喪地圖、找咖小幫手
    2 B2C 電商競品比價 電商 水果價格預測
    3 數位金融客訴效率 銀行 / FinTech 反詐騙 LINE Bot、打造自己的 ETF
    4 音樂串流推薦優化 媒體 / 串流 KKBOX&Spotify 比較、SteamTrend
    5 叫車服務交通熱點 交通 / 物流 國道車流 LSTM、國旅推薦
    6 求職媒合職缺趨勢 HR Tech 求職 360°、求職求值

    配套課堂材料¶

    這份手冊跟你在課堂上看到的三份簡報是一套，互相補強：

    階段 課堂簡報 對應手冊章節
    課程開始 課程總覽 & DE 介紹 (PDF) README + 04 痛點地圖
    前置期 W1-W3 期初工作坊：資料解決方案顧問 (PDF) 02 通用管線藍圖 + scenarios/
    實作期 W10-W12 期中工作坊：個人 pipeline 實作 (HTML) 03 10 階段流程 + 05 協作 SOP

    💡 建議閱讀順序：先看課堂簡報感受全貌，回手冊查細節與作業。

    記住心法¶

    「業界專案不是把技術秀出來，是把情境、痛點、解法(最優解和最快解)、遇到的技術問題和如何處理等講清楚。」

    手冊閱讀方法¶

    依你目前的狀態，找到適合的閱讀路徑：

    你的狀態 🎯 必讀（先看） 📖 下一步
    🌱 完全沒用過 GitHub 01 GitHub 註冊 SOP 05 協作 SOP
    🤝 還沒組好隊 / 不知怎麼啟動 03 10 階段流程（① 啟動段） 00 敏捷與工具選型
    🎯 組好了、正在選題 04 痛點 × 業界地圖 scenarios/ 挑 1-2 個對齊
    🏗 開始實作、不知如何設計 02 通用管線藍圖 對應的 scenario 文件
    🛠 動工中卡關 / 流程亂 05 協作 SOP + 00 敏捷與工具 03 10 階段流程（⑥ 衝刺段）
    🎤 快到 demo 了 07 Demo Day 評分表 06 復盤模板（寫 final retro）
    📜 專題完了想升級履歷 06 課後加強路徑 重看 07 評分表 self-check
    👀 第一次來、想全覽 從本 README 看完目錄 → 看 1 部 學長姐 demo 03 10 階段流程
