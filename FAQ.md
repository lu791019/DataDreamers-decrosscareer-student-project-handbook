# 學員常見問題 FAQ

## Q1：手冊寫很多，不太能理解，要如何開始跑專案的整體流程？

會覺得「好多」是正常的！那份手冊是設計成「字典」用的——不用一次讀完，而是到了那個階段再翻對應的章節。

### 整個專案其實就 3 大段

```
前置期：架構練習(期初) → 組隊 → 選題 → 準備Github → 確認溝通工具和專案管理工具
實作期：開始專題建構，每週分配任務並完成
收尾：整合 → 優化 → Demo → 演練 & 問題討論和確認
```

### 在前置期能做的：

**第一件：把 GitHub 搞定**

看 [01 GitHub 註冊 SOP](01-github-setup-sop.md)，照步驟和作業走：註冊 → 裝 git → 確認能使用 → 建議個人 Github 和團隊 Github。另外裡面有推薦 3 份學習資源，挑一份看就夠

**第二件：跟組員把工具選好**

看 [00 敏捷開發 + 工具選型](00-agile-and-tools.md) 最後面的「作業」，跟組員一起決定：溝通用什麼（Discord / LINE ？）、進度追蹤用什麼（Trello / Notion ？），然後把工具開起來

這兩件做完，你的專案用工具就完成了

### 專有名詞不用現在全懂

第 0 週提到的 Scrumban、WIP limit、Sprint Planning 那些，**不用現在全部理解**。先知道「每週跟組員對一次進度、用看板追任務」這個概念就好，跑起來之後自然會理解為什麼要那樣做。

### 怎麼知道接下來該看哪篇？

[README 最下面有一張「手冊閱讀方法」的表](README.md#手冊閱讀方法)，找到你目前的狀態，它會告訴你該先看哪篇、下一步看哪篇

### 一句話總結

**先裝工具 → 組隊選題 → 開始動手 → 卡關再回來查手冊。不用先讀完才開始跑。**
