# 學員常見問題 FAQ

> 收錄學員在課程中實際提出的問題與回答。遇到卡關先來這裡找找看。

---

## Q1：手冊寫很多，不太能理解，要如何開始跑專案的整體流程？ (原話：看了第0週的git專案的整體流程，老師寫的很多，可是我好像不太能理解，要如何開始跑專案的整體流程

這份手冊比較好的使用方式是：先知道現在自己在哪個階段，接著只看這個階段需要用到的章節就好

整個專案流程可以先簡化成 3 大段：

**第一段：前置期**
架構練習 → 組隊 → 選題 → 準備 GitHub （個人和團隊） → 確認溝通工具與專案管理工具

**第二段：實作期**
開始進行專題建構，每週和組員分配任務、追蹤進度，並逐步完成專案內容

**第三段：收尾期**
專案整合 → 功能優化 → Demo 準備 → 演練 → 問題討論與確認

所以一開始不用急著把所有東西都看懂。前置期最重要的是先完成兩件事和開始一件事：

### 完成第一件事：先把 GitHub 建好

可以先看 [01 GitHub 註冊 SOP](01-github-setup-sop.md)，照著裡面的步驟完成：

- 註冊 GitHub
- 安裝 Git
- 確認 Git 可以正常使用
- 建立個人 GitHub 與團隊 GitHub

裡面也有推薦幾份學習資源，但不用全部看完。先挑一份你看得懂、跟得上的資源即可

### 完成第二件事：跟組員一起把工具選好

可以看 [00 敏捷開發 + 工具選型](00-agile-and-tools.md) 最後面的作業，和組員一起討論：

- 溝通工具要用什麼？例如 Discord、LINE
- 進度追蹤工具要用什麼？例如 Trello、Notion
- 工具開好之後，大家要怎麼更新任務與同步進度？

這兩件事完成後，基本上你們專案前期需要的協作工具就算準備好了。

### 開始的一件事：邊學邊想團隊專題的題目

選題不是到第 4 週才「突然決定」的，而是從第 2 週開始，一邊看章節學、一邊跟組員慢慢聊出來的。做完前兩件事之後，就可以邊學邊想：

- **第 2 週（學 6 階段藍圖時）**：邊看邊想「這套骨架可以套在什麼題目上？」先發散，跟組員丟出 3-5 個可能方向
- **第 3 週（排時程、分工時）**：把方向收斂到 1-2 個，評估「資料拿不拿得到、團隊做不做得完」
- **第 4 週（對痛點地圖時）**：用 8 情境的業界痛點與職位對照，把題目正式定下來

換句話說，第 2~4 週的「學章節」和「想題目」是**同時進行**的。等到第 4 週結束，你們手上應該已經有一個定案題目 + 一份 persona／痛點文件，就能直接接進環境準備與實作期，不會卡在「還沒決定要做什麼」。

至於第 0 週提到的 Scrumban、WIP limit、Sprint Planning 等名詞，現在不用急著全部理解。剛開始只要先抓住一個核心概念就好：

> 每週跟組員對一次進度，並用看板追蹤每個人正在做什麼。

等到專案真的跑起來，你會比較自然地理解為什麼需要這些方法和名詞。

如果不知道接下來該看哪一篇，可以回到 [README 最下面的「手冊閱讀方法」表格](README.md#手冊閱讀方法)。找到你目前的狀態，它會告訴你現在應該先看哪一篇，以及下一步可以接著看哪一篇。

簡單來說，不需要先把整份手冊讀完才開始做專案。你可以先照這個順序開始：

1. 先選好團隊
2. 再確認和建置各種工具 <-現在在這
3. 討論選題 
4. 接著開始動手做
5. 遇到卡關時，再回來查手冊對應的章節

---

### 那前置期接下來幾週呢？（想先看清楚節奏的人再往下）

把上面兩件事做完之後，前置期每一週都對應一篇章節，照著走就好——一樣**不用提前讀完，到那一週再翻對應的章節**。重點是「每週往前推進一個產出」，而不是把手冊讀完。

| 週次 | 看哪篇 | 那一週要做什麼（產出） |
|---|---|---|
| **第 0 週** | [00 敏捷 + 工具選型](00-agile-and-tools.md) | 組隊、把溝通／專案管理工具開起來 |
| **第 1 週** | [01 GitHub SOP](01-github-setup-sop.md) | 帳號 + repo 設定好、能正常 push |
| **第 2 週** | [02 通用管線藍圖](02-pipeline-blueprint.md) | 學會「6 階段骨架」（Source → Ingest → Storage → Process → Serve → Observe），並可以依據期初活動或團隊選出的主題來**畫出你們專題的 6 階段架構圖**（篇末小作業） |
| **第 3 週** | [03 小組 10 階段流程](03-team-process.md) | 看懂從啟動到復盤的 20 週節奏，對照目前進度，**排出你們組的時間表、分好角色** |
| **第 4 週** | [04 痛點 × 業界地圖](04-pain-points-industry-map.md) | 對照 8 情境的業界痛點與職位，**把題目選定**，再去 [scenarios/](scenarios/) 挑 1-2 個對齊，**寫一份 persona + 痛點文件**（篇末小作業） |

**怎麼進行的小訣竅**：每篇章節最後幾乎都有一個 「小作業」— 那個作業就是「這一週的產出」。把小作業做完，這一週就算過關，自然接到下一週

> 第 2~4 週做完，你們就完成了「畫架構 → 安排之後團隊的開發工具和時程規劃 → 選題」三件事。之後（第 5 週起）進入環境準備，再接著進入實作衝刺期（約 W10 起）。實作期天天看 [05 協作 SOP](05-collaboration-sop.md) 就好；快到 demo 時再看 [07 Demo Day 評分表](07-demo-day-rubric.md)
