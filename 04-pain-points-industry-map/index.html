    專案協作第 4 週 ｜ 04 8 情境痛點 × 業界對應地圖¶

    每個情境一張卡：技術痛點 / 業務痛點 / 業界真實職位 / 你能在面試怎麼講。

    選題前先讀這一篇 — 你不是在選「好玩的題目」，是在選「想去的業界」。

    怎麼用這張地圖¶

    1. 看你想去的產業 → 找對應的情境
2. 看「業界職位」欄 → 確認那行有沒有興趣
3. 看「面試 5 秒講法」→ 想像你能不能講得自然
4. 都 OK → 跳去 scenarios/scenario-{N}-*.md 開始走

    情境 1：餐飲連鎖 POS¶

    業務問題：門市庫存判斷 + 補貨建議；熱銷分析助行銷

    維度 內容
    🔥 技術痛點 POS 資料髒（時間格式 / 商品代碼不一致）／ 多門市 schema 差異 ／ 即時性 vs 批次
    💼 業務痛點 老闆要的是「今天該不該叫貨、叫多少」，不是月報
    🏢 業界職位 餐飲集團（瓦城 / 王品 / 全家 / 統一）DE / 數據分析師、第三方 POS SaaS（iCHEF）DE
    🎯 過去學長姐 08 美味餐廳隨拾取 · 18 食喪地圖 · 20 找咖小幫手 · 16 水果價格預測
    💬 面試 5 秒講法 「我做了一個多門市 POS 庫存儀表板，每天自動產出補貨建議，預測庫存到底的時間。」
    🚀 進階展望 dbt + BigQuery 維度建模 / LLM 評論分析 / 即時警示
    ⚠️ 雷區 不要單純做「銷售報表」 — 太薄；要做「決策建議」

    情境 2：B2C 電商競品比價 + 動態定價¶

    業務問題：競品同款商品價格監控 + 自家定價策略

    維度 內容
    🔥 技術痛點 反爬蟲（cookie / 動態 JS / 速率限制）／ 商品 SKU 對齊 ／ 價格波動的時序處理
    💼 業務痛點 老闆要的是「我的價格在市場排第幾、要不要調」
    🏢 業界職位 momo / PChome / 蝦皮 / Yahoo 購物 DE、品牌商（家樂福 / 全聯）數據組
    🎯 過去學長姐 16 水果價格預測系統
    💬 面試 5 秒講法 「我每天爬 5 個電商平台同款商品，做出價格 dashboard 給品牌決定要不要跟降。」
    🚀 進階展望 CDC（Debezium）即時 / ML 自動定價 / 預測競品下次降價時間
    ⚠️ 雷區 爬蟲合規問題要看 robots.txt + 不要爬太狠（你不是要做 DDoS）

    情境 3：數位金融客訴效率 + 滿意度¶

    業務問題：客訴文字自動分類 + SLA 警示 + 滿意度量化

    維度 內容
    🔥 技術痛點 中文 NLP / 情緒分析 ／ 客訴量大時的批次處理 ／ 個資去識別化
    💼 業務痛點 老闆要的是「客訴量上升中嗎、是哪類客訴、誰要負責」
    🏢 業界職位 純網銀（將來 / LINE Bank）DE、傳統銀行數位金融部、保險科技、FinTech 新創
    🎯 過去學長姐 07 反詐騙 LINE Bot · 06 ETF
    💬 面試 5 秒講法 「我做了一個用 LLM 自動分類客訴的 pipeline，客服 SLA 超過閾值會推播警示。」
    🚀 進階展望 Vector DB + RAG 智能客服 / Kafka 串流警示
    ⚠️ 雷區 不要用 mock 資料 — 用 CFPB Complaint Database（美國消費者金融保護局公開）

    情境 4：音樂串流推薦優化 + 用戶偏好¶

    業務問題：用戶分群、推薦清單個人化、新歌推送

    維度 內容
    🔥 技術痛點 Spotify API rate limit / 用戶分群算法 ／ 即時 vs 離線推薦
    💼 業務痛點 老闆要的是「我的用戶留多久、聽多少、什麼能讓他們多停留」
    🏢 業界職位 KKBOX / Spotify / Apple Music DE、Podcast 平台、串流影音（Netflix / iQiyi）
    🎯 過去學長姐 03 KKBOX & Spotify 比較 · 01 SteamTrend
    💬 面試 5 秒講法 「我用 Spotify API 抓 50 萬筆聽歌紀錄，做出『跟你最近聽的歌氛圍類似但你還沒聽過』的推薦清單。」
    🚀 進階展望 Vector DB 推薦 / 協同過濾 + ML / Spark MLlib
    ⚠️ 雷區 不要做「最熱門 top 100」— 那不是推薦是排行榜

    情境 5：叫車服務交通熱點 + 車輛調度¶

    業務問題：時段供需熱點、即時推送司機去哪、預測尖峰

    維度 內容
    🔥 技術痛點 地理空間資料（lat/lng / geohash）／ 時序預測 ／ 即時 streaming 處理
    💼 業務痛點 老闆要的是「司機被叫不到的時段在哪、要不要加價」
    🏢 業界職位 Uber / LINE TAXI / 大車隊 DE、物流（lalamove / Pinkoi 物流組）、外送（foodpanda / UberEats）
    🎯 過去學長姐 13 國道車流 LSTM · 11 國旅推薦
    💬 面試 5 秒講法 「我用 NYC 計程車資料做了一個熱點預測模型，能提前 30 分鐘預測哪個區會缺車。」
    🚀 進階展望 Kafka + Spark Streaming 真即時 / ML 需求預測
    ⚠️ 雷區 地理資料計算密集 — 不要在 pandas loop，用 PostGIS / BigQuery GIS

    情境 6：求職媒合職缺趨勢 + 薪資洞察¶

    業務問題：職缺市場熱度、技能熱門排行、薪資中位數

    維度 內容
    🔥 技術痛點 104 / CakeResume 爬蟲反制 ／ 職缺描述 NLP 抽技能 ／ 薪資範圍 parsing
    💼 業務痛點 求職者要的是「我學這個技能薪水多少、市場缺多少人」
    🏢 業界職位 104 / 1111 / CakeResume DE、HR Tech 新創、企業 HR 數據組、求職 SaaS（Hahow）
    🎯 過去學長姐 04 求職 360° · 21 求職求值
    💬 面試 5 秒講法 「我爬了 104 三萬筆 DE 職缺，做了一個技能熱度榜 + 薪資中位數 dashboard。」
    🚀 進階展望 Embedding + Vector DB 履歷匹配 / 知識圖譜 / LLM 自動寫履歷
    ⚠️ 雷區 104 反爬蟲很硬 — 用 selenium / playwright + sleep 不要太快

    情境 7：媒體業新聞輿情即時監測¶

    業務問題：品牌 / 議題 / 政治輿情自動監測 + 警示

    維度 內容
    🔥 技術痛點 RSS / NewsAPI 整合 ／ 中文文章分類 + 情緒 ／ 主題追蹤（topic modeling）
    💼 業務痛點 客戶要的是「我的品牌今天被罵了嗎、誰在罵、要不要回應」
    🏢 業界職位 公關公司（精誠 / 凱絡）DE、媒體（聯合 / 中天）、行銷代理（dentsu / cacaFly）
    🎯 過去學長姐 05 社群輿情 · 10 夜市水軍王 · 19 遊戲輿情 · 22 巴哈姆特輿情 · 02 NASA Line Bot
    💬 面試 5 秒講法 「我做了一個用 LLM 分析品牌輿情的 pipeline，每小時自動掃 PTT/Threads/新聞，負面情緒爆量就推播。」
    🚀 進階展望 Kafka 即時 / Vector DB 主題追蹤 / 自動 generate 回應稿
    ⚠️ 雷區 LLM cost 控制 — 不要每篇文章都用 GPT-4，先用便宜的篩

    情境 8：不動產房價趨勢分析¶

    業務問題：區域房價走勢、預測未來價、買賣決策建議

    維度 內容
    🔥 技術痛點 實價登錄 zip 解析 ／ 地理座標補齊 ／ 時序預測模型
    💼 業務痛點 投資人 / 自住者要的是「這區域明年漲跌、現在進場合不合理」
    🏢 業界職位 永慶 / 信義房屋 / 591 DE、PropTech 新創、銀行房貸部、地產投資公司
    🎯 過去學長姐 14 房價走勢密碼 · 15 六都寵物便利度
    💬 面試 5 秒講法 「我用實價登錄 + 周邊設施做了一個房價預測模型，輸入地址就能看區域 3 年走勢 + 投資評分。」
    🚀 進階展望 PostGIS 地理索引 / ML 房價預測 / dbt + BigQuery
    ⚠️ 雷區 不要只做「平均房價地圖」 — 地圖到處都有，重點是「趨勢預測 + 解釋」

    跨情境共通的 5 個業界痛點¶

    不管選哪題，業界永遠在解這 5 件事 — 你做的時候有對到，面試就有得講：

    # 共通痛點 怎麼在你的專案展示
    1 資料品質 加 dbt tests / Great Expectations，dashboard 上顯示「資料健康度」
    2 管線可靠性 Airflow / cron + 失敗推播 + retry 機制
    3 資料新鮮度 dashboard 顯示「最後更新時間」+ 過期警示
    4 Cost 控制 算過 cloud bill、知道哪段最貴、有優化過
    5 權限 / 個資 至少做了基本去識別化、敏感資料不入 git

    怎麼選題（決策樹）¶

    你想去哪個產業？
├ 餐飲 / 零售           → 1 餐飲 POS
├ 電商 / B2C            → 2 比價 / 動態定價
├ 金融 / 銀行 / 保險    → 3 金融客訴
├ 媒體 / 內容 / 串流    → 4 音樂 / 7 輿情
├ 交通 / 物流 / 外送    → 5 叫車交通
├ HR / 教育             → 6 求職媒合
├ 行銷 / 公關 / 廣告    → 7 新聞輿情
├ 不動產 / PropTech     → 8 房地產
└ 都行 / 不確定         → 看哪題你「過去職涯」最有故事

    教練建議：選一個「跟你過去工作 / 生活最有連結」的題目。 你做電商，做 2 比做 4 強；你聽很多歌，做 4 比做 2 強。 故事比技術重要。

    🎯 小作業：寫一份 persona + 痛點文件¶

    挑好情境後，把「我們是為誰解決什麼」寫具體。

    • 寫 1 個具體 persona（姓名 / 年齡 / 職業 / 月薪 / 生活情境 / 一句話痛點，不准寫「年輕人」這種模糊版）
    • 列 3-5 個可量化痛點（例：「每天花 30 分鐘做 X」「每月損失 Y 元」）
    • 寫成 docs/persona.md commit 進 repo
    • 寫一句話：「我們在課程結束時想讓這個 persona 從 A 變成 B」

    交付：PR 連結貼到指定 channel。

    💡 卡關時看 03-team-process.md 的 Persona 範例（具體 vs 模糊），或對應的 scenario 文件「為誰解什麼」段。
