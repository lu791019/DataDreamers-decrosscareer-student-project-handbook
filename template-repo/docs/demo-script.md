### 1:30 - 3:30｜Live Demo

> 「現在我們現場跑一次。」

**Demo 順序**（事先排練）：
1. 打開 dashboard → 顯示「現在狀態」
2. 點 X 功能 → 顯示 Y 結果
3. 模擬一個情境（新事件進來）→ 自動處理
4. 切換到另一個視圖 → 顯示對比

**講者**：@product_owner（操作）+ @analytics_owner（解釋）

**Demo 翻車備案**：
- 如果即時資料抓不到 → 切到「歷史模擬」按鈕
- 如果 network 掛 → 切到本地版（已準備）
- 螢幕錄影備份（最壞情況播影片）

## 共通 Tips

### 排練 checklist
- [ ] 組內 dry run 至少 3 次
- [ ] 計時：每段不超過預算
- [ ] Live demo 至少 dry run 5 次（最容易翻車）
- [ ] 螢幕錄影備份
- [ ] 投影片 PDF 備份（網路掛了還能講）

### 5 個常見 Q&A 你要準備

1. **「為什麼選這題？」**
   → 從 persona + 你個人故事切入

2. **「為什麼用 X 不用 Y？」**
   → 用 ADR 的邏輯回答（context / trade-offs）

3. **「踩過最大的坑？」**
   → 講過程 + 學到的、不要只講結果

4. **「如果有 10x 資料量？」**
   → 用架構圖那段的擴展性表

5. **「組員分工是怎樣？」**
   → 講 RACI、不要說「平均分」

---

## 投影片建議

- **頁數 ≤ 10**
- 第 1 頁：title + team + 1 句話
- 第 2 頁：persona / problem
- 第 3 頁：方案 overview（一張圖）
- 第 4-7 頁：架構 / demo / 影響
- 第 8 頁：踩過的坑
- 第 9 頁：未來
- 第 10 頁：Thank you + QR code 連結

字體 ≥ 24pt；每頁字 ≤ 50；多用圖少用字。
